OfficeExtender Overview
If your using the Office2003,Office2007 or MediaPlayer visual styles in your application when you will probably want to make use of this handy little component. Unlike other visual style settings the office styles do not use the standard control background color. So if you want your Form / Panel or Control derived classes to have the same background color as the current office appearance you need to take additional steps.

This is where the OfficeExtender control comes into use. It can be used to automatically update the background color of specified control instances with the correct color background value. This will take into account the current desktop theme if using the Office2003 style as well as the requested variation of the color needed.

Design Time

The easiest way to use the extender control is to drag and drop an instance onto your Form at design time. As an extender type control it will then add two new properties to each of the child controls on your design surface. These are called Office2003BackColor and Office2007BackColor and cause the BackColor property on the target control to be automatically set according to the selected enumeration value. The possible values for both of the extender properties is as follows: -
  • Disable
  • Base
  • Light
  • Dark
  • Enhanced
The default value is Disable and will prevent the background color being set automatically. If you choose any of the other values than the appropriate color variation for the desktop theme will be set. When the desktop scheme is changed by the user at runtime then any required change will also be proprogated to the controls.

Programmatic Use

In many situations you will not be able to use the extender at design time. Maybe you are creating instances of controls at runtime dynamically or loading information from a cofiguration file. But you can still take advantage of the extender control by creating an instance in code and then calling the appropriate methods.

Indeed, this is exactly what the docking manager class does in the library in order to automatically update Form and Panel instances that are added as Content at runtime. Here is an example of creating an instance and adding two controls that will be updated with the Base and Dark theme color variations for Office2003 styling : -

C#


   OfficeExtender extender = new OfficeExtender();
	
   extender.SetOffice2003BackColor(_control1, OfficeColor.Base);
   extender.SetOffice2003BackColor(_control2, OfficeColor.Dark);
VB.NET

   Dim extender As New OfficeExtender()
	
   extender.SetOffice2003BackColor(_control1, OfficeColor.Base)
   extender.SetOffice2003BackColor(_control2, OfficeColor.Dark)
If instead you are using one of the three Office2007 styles then you would need to hook into the Office2007BackColor extender property instead. You also need to set the Office2007Variant property to the correct color scheme that you are using for Office2007. Here is the alternative : -


   OfficeExtender extender = new OfficeExtender();
	
   extender.Office2007Variant = Office2007Variant.Silver;
   extender.SetOffice2007BackColor(_control1, OfficeColor.Base);
   extender.SetOffice2007BackColor(_control2, OfficeColor.Dark);
VB.NET

   Dim extender As New OfficeExtender()
	
   extender.Office2007Variant = Office2007Variant.Silver
   extender.SetOffice2007BackColor(_control1, OfficeColor.Base)
   extender.SetOffice2007BackColor(_control2, OfficeColor.Dark)
If your application changes to a different color scheme that is still Office2007 then you would only need to update the Office2007Variant property to the new scheme.

You can use the GetOffice2003BackColor / GetOffice2007BackColor methods to discover the current settings of a control that has already been added to the extender. Plus the RemoveControl method is useful when you are dynamically removing controls from your Form and so also need it removed from the extender.

Here is an example that shows the methods being used: -

C#


   Console.WriteLine("Control 2003 Setting: {0}",
      extender.GetOffice2003BackColor(_control1));

   Console.WriteLine("Control 2007 Setting: {0}",
      extender.GetOffice2007BackColor(_control1));
      
   extender.RemoveControl(_control1);
VB.NET

   Console.WriteLine("Control 2003 Setting: {0}", _
      extender.GetOffice2003BackColor(_control1))
      
   Console.WriteLine("Control 2007 Setting: {0}", _
      extender.GetOffice2007BackColor(_control1))

   extender.RemoveControl(_control1)
The three MediaPlayer styles work in the same manner as the Office2007 styles and so you should use the following code to set the MediaPlayerVariant property to the correct color scheme.

C#


   OfficeExtender extender = new OfficeExtender();
	
   extender.MediaPlayerVariant = MediaPlayerVariant.Orange;
   extender.SetMediaPlayerBackColor(_control1, OfficeColor.Base);
   extender.SetMediaPlayerBackColor(_control2, OfficeColor.Dark);
VB.NET

   Dim extender As New OfficeExtender()
	
   extender.MediaPlayerVariant = MediaPlayerVariant.Orange
   extender.SetMediaPlayerBackColor(_control1, MediaPlayerColor.Base)
   extender.SetMediaPlayerBackColor(_control2, MediaPlayerColor.Dark)
NOTE that you should remember to call the Dispose method on the extender just like any other control when you have finished using it. Otherwise extra references to controls will persist and garbage collection will not work as expected.
