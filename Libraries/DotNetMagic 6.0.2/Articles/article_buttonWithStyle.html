ButtonWithStyle Overview
Your application may need the ability to show a button control with the same style as you have selected for other DotNetMagic controls. If your are using a ToolStrip or the MenuStrip with the VisualStyle.Office2003 style you might prefer to use a button that has the same style of appearance. Use the SampleButtonWithStyle sample application in order to alter the control properties and see the effect on the control.

The following sections show how to use the control and describe the more important properties and events for controlling the ButtonWithStyle. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Appearance

Style property

This property is used to determine the look and feel of the button control. This property is present on most of the DotNetMagic controls and allows a consistent look and feel to be achieved across your whole application. The nine possible values for the property are: -
  • VisualStyle.Plain
  • VisualStyle.IDE2005
  • VisualStyle.Office2003
  • VisualStyle.Office2007Blue
  • VisualStyle.Office2007Silver
  • VisualStyle.Office2007Black
  • VisualStyle.MediaPlayerBlue
  • VisualStyle.MediaPlayerOrange
  • VisualStyle.MediaPlayerPurple
VisualStyle.Plain will not draw a border around the button until the mouse moves over the button area and then a border is drawn.

VisualStyle.IDE2005 will draw with the look and feel of the VS.NET 2005 environment.

VisualStyle.Office2003 will draw the button with the same style of gradient effect and colors as the standard Microsoft Office 2003 applications.

VisualStyle.Office2007Blue will draw with the Blue color scheme from Office 2007. VisualStyle.Office2007Silver will draw with the Silver color scheme from Office 2007. VisualStyle.Office2007Black will draw with the Black color scheme from Office 2007. VisualStyle.MediaPlayerBlue will draw in a Blue variation of the Windows Media Player. VisualStyle.MediaPlayerOrange will draw in a Orangevariation of the Windows Media Player. VisualStyle.MediaPlayerPurple will draw in a Purple variation of the Windows Media Player.

Office2003GradBack property

This property is only used when the VisualStyle.Office2003 setting is defined for the Style property. When defined as False the control background will be a solid color relevant to the current theme. When True the background will be a gradient from a darker color on the left to a lighter color on the right. This is used to match the color change that can be seen with Microsoft 2003 suite applications on the menu bar area.

IDE2005GradBack property

This property is only used when the VisualStyle.IDE2005 setting is defined for the Style property. When defined as False the control background will be a solid color relevant to the current theme. When True the background will be a gradient from a darker color on the left to a lighter color on the right.

StaticIDE property

This property is only used when the VisualStyle.IDE setting is defined for the Style property. When defined as False the button background works in the same manner as the menu and toolbar. So when the user tracks the mouse over the button any image is shown with a shadow effect. When defined as True the image is not shown with a shadow.

ButtonStyle and Pushed properties

The two possible values for the property are: -
  • ButtonStyle.PushButton
  • ButtonStyle.ToggleButton
PushButton style will ignore the values of the Pushed property and so act as a simple button at all times. Each time the button is pressed the Click event is generated.

ToggleButton style will draw the control according to the value of the associated Pushed property. Each time the button is pressed the value of Pushed is inverted and the Click event generated.

AlwaysDrawEnabled

By default this property is false but when defined it will draw the button with an enabled appearance even when the button control has been disabled.

AlwaysDrawBorder

By default this property is false but when defined it will draw a border around the button to ensure the button outline can be seen. This is useful when using visual styles such as IDE2005 where a border is not usually drawn unless the mouse is hovering over the button.

DrawBackground

By default this property is true and so the background of the button is always drawn. You can however set this to false if you want the parent background to show through when the mouse is not over the button and the button has an image that has some transparency.

Image and Text properties

You can use the standard Text property to assign some display text that will be drawn on the face of the button. Alternatively you could assign a loaded image to the Image property and show a picture instead. If you need both text and image then you just assign to both of the properties. When this happens the TextEdge property is used to determine how to arrange the two elements.

TextEdge property

This property will only affect the appearance of the control when both the Text and the Image properties have been assigned valid values. Only in this situation does the control need to make a decision about how to arrange the two visual requirements.

The four possible values for the property are: -

  • TextEdge.Top
  • TextEdge.Bottom
  • TextEdge.Left
  • TextEdge.Right
TextEdge.Top will place the text in the upper part of the control and the image positioned below the text.

TextEdge.Bottom will place the image in the upper part of the control and the text positioned below the image.

TextEdge.Left will place the text in the left part of the control and the image positioned to the right of the text.

TextEdge.Right will place the image in the left part of the control and the text positioned to the right of the image.

Direction property

Use this property to determine in what direction the contents of the button should be displayed.

The two possible values for the property are: -

  • Direction.Horizontal
  • Direction.Vertical
Direction.Horizontal is the default and displays the text in the standard left to right direction.

Direction.Vertical is used to show the text drawn in a vertical manner. It effectively rotates the control to the right and is useful for placing several instances one above the other and reducing the horizontal space required.

TextAlignH

Use this property of type StringAlignment to decide how the text is positioned horizontally within its allocated area of the button. Note that this is most effective when only text is shown inside the button instance.

TextAlignV

Use this property of type StringAlignment to decide how the text is positioned vertically within its allocated area of the button. Note that this is most effective when only text is shown inside the button instance.
Notifications

Click event

When the user presses and then releases the button it will generate the Click event. This works in the same manner as the existing Button class that is provided from the Windows.Forms namespace. Note that you can also simulate the pressing of the button by calling the PerformClick method.
Sample code
C# 
	
   // Create a new button instance
   ButtonWithStyle myButton = new ButtonWithStyle();

   // We want to ensure the Office 2007 style
   myButton.Style = VisualStyle.Office2007Blue;
   
   // We want it to act as a toggle button
   myButton.ButtonStyle = ButtonStyle.ToggleButton;
   
   // Start by being pushed
   myButton.Pushed = true;
   
   // Set the button text
   myButton.Text = "Test";

   // Define the size and position
   myButton.Size = new Size(64,64);
   myButton.Location = new Point(10,10);

   // We want notification when button is pressed
   myButton.Click += new EventHandler(OnButtonClick);

   Controls.Add(myButton);
VB.NET 

   ' Create a new button instance
   Dim myButton As New ButtonWithStyle()

   ' We want to ensure the Office 2007 style
   myButton.Style = VisualStyle.Office2007Blue
   
   ' We want it to act as a toggle button
   myButton.ButtonStyle = ButtonStyle.ToggleButton
   
   ' Start by being pushed
   myButton.Pushed = true
   
   ' Set the button text
   myButton.Text = "Test"

   ' Define the size and position
   myButton.Size = New Size(64,64)
   myButton.Location = New Point(10,10)

   ' We want notification when button is pressed
   AddHandler myButton.Click, AddressOf OnButtonClick

   Controls.Add(myButton)
