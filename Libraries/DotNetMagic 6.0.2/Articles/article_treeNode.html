Node Overview
The Node class represents the basic building block used to create a complex hierarchy that can be displayed using the TreeControl. You should also read the article that describes the NodeCollection class by using the following link NodeCollection article. Plus ensure you read the main TreeControl article using this link TreeControl article.

The following sections show how to use Node class and describe the more important properties, methods and events for controlling the Node. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Appearance
Text
Specifies the text that will be shown for this node instance.

Tooltip
Text to show as a tooltip when the user hovers the mouse over this node instance. This will only be shown if the TreeControl.Infotips property is defined as True and if the Tooltip text is at least one character in length.

NodeFont
The default value of null will cause the Font from the TreeControl to be used when drawing the node text. If a specific Font instance is assigned to this property then it will be used instead of inheriting from the TreeControl.

ForeColor
The default value of Color.Empty will cause the ForeColor from the TreeControl to be used when draw the node text. If a specific color is assigned to this property then it will be used instead of inheriting from the TreeControl.

BackColor
The default value of Color.Empty will cause the BackColor from the TreeControl to be used when drawing the background for the text area. If a specific color is assigned to this property then it will be used instead of inheriting from the TreeControl.

ImageIndex
The default value of -1 will cause the ImageIndex from the TreeControl to be used when drawing an image from the ImageList of the TreeControl. Note that this is used when the node is not in the selected state, in which case the SelectedImageIndex property is used instead. If a specific index is assigned to this property then it will be used instead of inheriting from the TreeControl.

SelectedImageIndex
The default value of -1 will cause the SelectedImageIndex from the TreeControl to be used when drawing an image from the ImageList of the TreeControl. Note that this is only used when the node is in the selected state, otherwise the ImageIndex is used instead. If a specific index is assigned to this property then it will be used instead of inheriting from the TreeControl.

Icon
If you need to display an Icon instead of using an image then use this property. Note that it will only be shown when the node is not in the selected state, use the SelectedIcon property to define the image for when the node is selected. The Icon property takes higher precedance than the Image property which itself has a higher priority than an assigned ImageIndex.

SelectedIcon
If you need to display an Icon instead of using an image then use this property. Note that it will only be shown when the node is in the selected state, use the Icon property to define the image for when the node is not selected. The SelectedIcon property takes higher precedance than the SelectedImage property which itself has a higher priority than an assigned SelectedImageIndex.

Image
It might not always be possible or desirable to use an ImageList for storing all the images needed for your TreeControl. Each of the images might be different sizes and so cannot be placed into a single ImageList instance, plus you might not want every node to have space allocated for an image. In these cases you can assign an Image instance to this attribute. The Image property takes higher precedance than the ImageIndex property but is a lower priority than an assigned Icon.

SelectedImage
As with the Image property, the SelectedImage property takes higher precedance than the SelectedImageIndex property but is a lower priority than an assigned SelectedIcon. The SelectedImage is only used when the node is in the selected state.

Indicator
Just a few of the standard indicators provided are: -

  • None
  • Arrow
  • Box
  • Tick
  • Cross
  • Question
  • NoEntry
  • Exclamation
  • Error
  • Lock
Many of the entries in the above list are provided in a range of different colors. For example, there are red, orange, green, blue and gray versions of the arrow indicator. If a value of None is assigned then there will be no image drawn alongside the node, otherwise the appropriate picture is drawn.

CheckStates
The possible values for the enumeration property are: -

  • None
  • TwoStateCheck
  • ThreeStateCheck
  • Radio
  • Inherit
A value of Inherit will use the value from the CheckStates property of the TreeControl. If you explicitly provide a value of None then no checkbox or radio button is provided. Other values will cause the appropriate type of checkbox or radio button to be shown at the start of the node.

CheckState
The possible values for the enumeration property are: -

  • Unchecked
  • Mixed
  • Checked
The value of this property is used to decide how to draw the checkbox or radio button. It will also be updated with the new appropriate value when the user clicks on the checkbox/radio button to change its state.

Checked
This boolean property returns true if the CheckState is equal to Checked otherwise it returns false. It is provided for compatibility with the existing TreeNode class that is provided as part of the Windows.Forms namespace and makes it easier to change over from using the TreeView to the TreeControl.

Visible
You can prevent this node from being displayed as a part of the tree by setting this property to false. This will also prevent any of the children of the node from being shown. Using this property prevents the need to remove and then add back the node when you need to temporarily remove it from display.

IsVisible
This boolean property returns true if the Visible property is equal to true otherwise it returns false. It is provided for compatibility with the existing TreeNode class that is provided as part of the Windows.Forms namespace and makes it easier to change over from using the TreeView to the TreeControl.

Show()
Use this method as an alternative way to set the Visible property to true.

Hide()
Use this method as an alternative way to set the Visible property to false.

Expanded
When a node has children this property determines if the children should be displayed.

IsExpanded
This boolean property returns true if the Expanded property is equal to true otherwise it returns false. It is provided for compatibility with the existing TreeNode class that is provided as part of the Windows.Forms namespace and makes it easier to change over from using the TreeView to the TreeControl.

Expand()
Use this method as an alternative way to set the Expanded property to true.

ExpandAll()
This will not only set the Expanded state of the node to true but will also traverse all the child nodes in order to expand them as well.

Collapse()
Use this method as an alternative way to set the Expanded property to false.

CollapseAll()
This will not only set the Expanded state of the node to false but will also traverse all the child nodes in order to collapse them as well.

Toggle()
Use this method to invert the current state of the Expanded property.

Navigation
Nodes
A read-only property gives access to the collection of child nodes.

Parent
A read-only property that returns a reference to the Node instance that contains this instance. This can return a value of null if the node does not have a parent instance. Either because it is a root level node of the TreeControl or because it has been created but not yet added into another node.

ParentNodes
A read-only property referencing the NodeCollection that contains this instance. This can return a value of null if the node does not have a parent instance. Either because it is a root level node of the TreeControl or because it has been created but not yet added into another node.

TreeControl
A read-only property returning the reference to the TreeControl that this instance is a part of. Note that this can return null when the node is created but has not yet been added to a part of the tree hierarchy for a TreeControl.

Index
When the node instance has a parent node then this returns the index of the node within the parent collection of child nodes. Otherwise it will return -1.

Remove()
Use this method to remove the node from any parent collection.

GetNodeCount()
Use this method to return a count of the number of child nodes.

FirstNode
Use this method to return the first node in this section of hierarchy. This means it will always return a reference to itself as it represents the top.

FirstDisplayedNode
Use this method to return the first visible node in this section of hierarchy. If this node instance is visible then it will return a reference to itself, otherwise it will return null.

LastNode
Use this method to return the last node in the child hierarchy. It ignores the expanded or visible state of any node and returns the last node regardless of its current state.

LastDisplayedNode
Unlike the previous method this one will take into account the expanded and visible state of individual nodes. Therefore it will return the last node in the child hierarchy that is visible and will not traverse down a node that is not expanded.

NextNode
Returns the next node in the tree hierarchy. If the node has any children then it will return a reference to the first child, otherwise it will navigate up the chain until it finds a node that is not last in its collection.

NextDisplayedNode
Unlike the previous method this one will take into account the expanded and visible state of individual nodes.

PreviousNode
Returns the previous node in the tree hierarchy. If the node is not the first sibling then it will return the previous sibling otherwise it will return the parent node instance.

PreviousDisplayedNode
Unlike the previous method this one will take into account the expanded and visible state of individual nodes.

Others
Selectable
Use this property to specify if the node is allowed to be selected by the user.

IsSelected
This boolean property returns true if the node is currently part of the selection, otherwise it will return false.

Select()
Use this method to add the node to the current selection.

Deselect()
Use this method to remove the node from the current selection.

BeginEdit()
Call this method to start the editing process of the node text.

FullPath
A read-only property that returns the path used to navigate from the root to this node instance. It will use the TreeControl.PathSeparator property for deciding the separator string used between the names of each node in the path.

Key
This property may contain either null or a value that is unique within the set of node instances in the control. This allows the use of the GetNodeFromKey method on the control to be used to find a node instance based on the unique key value.

Tag
Provided so the developer can associate data to the node.

Size
A read-only property that returns the last calculated size needed to draw the node.

Bounds
A read-only property that returns the last calculated bounding rectangle for drawing the node.

ChildBounds
A read-only property that returns the last calculated bounding rectangle for the node that also includes all the child nodes as well.

VC
An advanced property used to assign a specific ViewController implementation for handling all the drawing and behavior of the node instance. By default this is null and so the TreeControl defined ViewController will be used instead.

ForeColorChanged
Generated when the ForeColor property changes value.

ImageIndexChanged
Generated when the ImageIndex property changes value.

SelectedImageIndexChanged
Generated when the SelectedImage property changes value.

IconChanged
Generated when the Icon property changes value.

SelectedIconChanged
Generated when the SelectedIcon property changes value.

ImageChanged
Generated when the Image property changes value.

SelectedImageChanged
Generated when the SelectedImage property changes value.

IndicatorChanged
Generated when the Indicator property changes value.

CheckStateChanged
Generated when the CheckState property changes value.

CheckStatesChanged
Generated when the CheckStates property changes value.

VisibleChanged
Generated when the Visible property changes value.

SelectableChanged
Generated when the Selectable property changes value.

ExpandedChanged
Generated when the Expanded property changes value.

TagChanged
Generated when the Tag property changes value.

VCChanged
Generated when the VC property changes value.
