TitleBar Overview
This handy little control can be used in combination with others to produce clear and consistent titles for your application. It has a large variety of settings so that its appearance and operation can be configured to exactly match your needs. To see how it can be combined with other controls you should try the SampleTabbedAdv example.

The following sections show how to use the control and describe the more important properties and events for controlling the TitleBar. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Properties

Style property

This property is used to determine the look and feel of the control. This property is present on most of the DotNetMagic controls and allows a consistent look and feel to be achieved across your whole application. The three possible values for the property are: -
  • VisualStyle.Plain
  • VisualStyle.IDE2005
  • VisualStyle.Office2003
  • VisualStyle.Office2007Blue
  • VisualStyle.Office2007Silver
  • VisualStyle.Office2007Black
VisualStyle.Plain will not draw a border around the button until the mouse moves over the button area and then a border is drawn.

VisualStyle.IDE2005 will draw with the look and feel of the VS.NET 2005 environment.

VisualStyle.Office2003 will draw the button with the same style of gradient effect and colors as the standard Microsoft Office 2003 applications.

VisualStyle.Office2007Blue will draw with the Blue color scheme from Office 2007. VisualStyle.Office2007Silver will draw with the Silver color scheme from Office 2007. VisualStyle.Office2007Black will draw with the Black color scheme from Office 2007.

Active property

There are times when you need to show the title bar in an inactive appearance. This allows your application to show one area as active and the others as inactive and so give the user a visual feedback. You might use this to show where the focus is or to show where the next user action is required.

Direction property

The two possible values for the property are: -
  • Direction.Horizontal
  • Direction.Vertical
When the vertical direction is selected the text is drawn from top to bottom in the control. It is also important to remember that the other appropriate properties are rotated in effect. So if you define GradientDirection to be from TopToBottom then when in vertical mode this will actually appear from right to left.

ActAsButton property

In some circumstances you will want the control to act like a button. Where the user can press the control in order to generate an event that your application can process. Use this property to define the style of button operation. The three possible values for the property are: -
  • ActAsButton.None
  • ActAsButton.JustArrow
  • ActAsButton.WholeControl
ActAsButton.None will prevent the control from acting like a button at all.
ActAsButton.JustArrow will only act as button when the arrow is pressed but other areas are ignored.
ActAsButton.WholeControl will act as button over the whole control area.

ArrowButton property

The five possible values for the property are: -
  • ArrowButton.None
  • ArrowButton.UpArrow
  • ArrowButton.DownArrow
  • ArrowButton.LeftArrow
  • ArrowButton.RightArrow
  • ArrowButton.Pinned
  • ArrowButton.Unpinned
ArrowButton.None will prevent the arrow from being drawn.
ArrowButton.UpArrow will draw the arrow pointing upwards.
ArrowButton.DownArrow will draw the arrow pointing downwards.
ArrowButton.LeftArrow will draw the arrow pointing left.
ArrowButton.RightArrow will draw the arrow pointing right.
ArrowButton.Pinned will draw a pin symbol horizontally..
ArrowButton.Unpinned will draw a pin symbol vertically.
Coloring

BackColor and ForeColor properties

These two colors are usually inherited from the parent control. In our case these are defaulted to use the system colors for the active caption. These make more sense as the standard control colors would not stand out as titles.

InactiveBackColor and InactiveForeColor properties

These two colors are used in place of BackColor and ForeColor when the control is set as being active. This is achieved by setting the Active property to False. By default these values use the system colors for an inactive caption.

GradientActiveColor and GradientInactiveColor properties

If you choose to draw with a gradient coloring then you need an extra target color. This source will be the BackColor or InactiveBackColor depending on the active state of the control. Define these two so that you achieve the required gradient effect.

GradientDirection property

The nine possible values for the property are: -
  • GradientDirection.None
  • GradientDirection.LeftToRight
  • GradientDirection.TopLeftToBottomRight
  • GradientDirection.TopToBottom
  • GradientDirection.TopRightToBottomLeft
  • GradientDirection.RightToLeft
  • GradientDirection.BottomRightToTopLeft
  • GradientDirection.BottomToTop
  • GradientDirection.BottomLeftToTopRight
If you prefer to have no gradient effect then set the value to be GradientDirection.None, otherwise a gradient will be drawn in the background using the direction as set. By default the VisualStyle.IDE setting will define this property as GradientDirection.TopToBottom and VisualStyle.Plain as GradientDirection.None.

GradientColoring property

The five possible values for the property are: -
  • GradientColoring.LightBackToBack
  • GradientColoring.LightBackToDarkBack
  • GradientColoring.BackToDarkBack
  • GradientColoring.BackToGradientColor
  • GradientColoring.LightBackToGradientColor
Use this property to define how the source and target colors for drawing a gradient background are calculated. Only the last two options will use the GradientActiveColor and GradientInactiveColor properties for the target.

MouseOverColor property

When the user moves the mouse over the control you might want the control to change the color of the text and arrow button. By default this property is defined as Color.Empty and so the color is not changed on the mouse moving over the control. Change this to a specific color to enable the functionality.
Contents

PreText, Text and PostText properties

The text that is displayed in the control is made up of three components. It is the aggregation of the PreText ,Text and PostText values added together. This is to make it easier when constructing titles for complex applications. The first part, the PreText might be the name of a project. Next you might have the Text defined as the actual name of the document. Finally the PostText could be a modifier to indicate if the document is dirty. By splitting these values at the TitleBar level the application code does not need to perform the aggregation itself each time one of the three components changes.

PreSeparator and PostSeparator properties

Rather than just concatenate the three text components together you can define an appropriate separator text to be placed between them. Typically this would be a '-' character as can be seen in the Visual Studio title. Note that the separators are only inserted if the relevant components are not empty. So if the PreText value is empty then the PreSeparator is also ignored. If the PreText has a defined value then it and the PreSeparator are concatenated and made part of the display string. Likewise for the PostText and PostSeparator properties.

Icon property

This property takes precendence over any specified Image or ImageList values that might also be set at the same time. Note that the Icon has a disadvantage over using an Image. When drawing in the inactive mode the Icon is still drawn in exactly the same way, but an Image will be drawn grayed out in an inactive appearance.

Image property

This property is of lower priority than the Icon but still used in preference to any ImageList and ImageIndex combination. When the control is marked as inactive the Image will be drawn washed out and inactive in appearance.

ImageList and ImageIndex

Provide both of these properties to use an ImageList for the source of an Image to be displayed.
Positioning

TextAlignment property

The three possible values for the property are: -
  • StringAlignment.Near
  • StringAlignment.Center
  • StringAlignment.Far
This controls the horizontal alignment of the text within the control.

LineAlignment property

The three possible values for the property are: -
  • StringAlignment.Near
  • StringAlignment.Center
  • StringAlignment.Far
This controls the vertical alignment of the text and any drawn Icon/Image within the control. Both the graphic and the text are always shown on the same line level.

ImageAlignment property

The two possible values for the property are: -
  • ImageAlignment.Near
  • ImageAlignment.Far
Use this property to define the position of the graphic relative to the text. If Near then the graphic will be placed on the left of the text and when define as Far on the right of the text line.

ArrowAlignment property

The two possible values for the property are: -
  • ArrowAlignment.Near
  • ArrowAlignment.Far
The arrow is always drawn either at the very start of the line, before any graphics or text, alternatively at the very end and so after any graphic and text.

PadLeft, PadRight, PadTop and PadBottom properties

You can modify these properties to adjust the exact position of text, graphic and arrow in the control. These are used to determine the padding around the edge of the control to restrict the available drawing area.

ImageToTextGap property

Used to define the number of pixels between any drawn graphic and the text that is next to it.

EdgeToImageGap property

Used to define the number of extra pixels between the graphics nearest edge and where it is drawn.

EdgeToArrowGap property

Used to define the number of extra pixels between the arrow and the nearest edge.
