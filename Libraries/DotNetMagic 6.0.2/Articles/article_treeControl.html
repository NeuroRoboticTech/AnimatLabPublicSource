TreeControl Overview
The TreeControl is an ideal way to show information that has a hierarchical nature. It allows easy access and display of information as well as a familiar and well understood metaphor for windows users. The fundamental building block for the data is the Node class that also acts as a container for a group of child Node instances. By placing Node instances inside other existing Node instances you can quickly create a tree hierarchy.

For more detailed information about the properties, methods and events of the Node class use the following link Node article. Each Node instance has a collection of child nodes and for more details on the collection class implementation use this link NodeCollection article. The following sections show how to use the control and describe the more important properties and events for controlling the TreeControl. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Styles
SetTreeControlStyle(TreeControlStyles style)

The possible values for the TreeControlStyles enumeration property are: -
  • StandardPlain
  • StandardThemed
  • Explorer
  • Navigator
  • Group
  • List
  • GroupOfficeLight
  • GroupOfficeDark
  • GroupOfficeBlueLight
  • GroupOfficeBlueDark
  • GroupOfficeSilverLight
  • GroupOfficeSilverDark
  • GroupOfficeBlackLight
  • GroupOfficeBlackDark
  • GroupMediaPlayerLight
  • GroupMediaPlayerDark
  • GroupMediaPlayerBlueLight
  • GroupMediaPlayerBlueDark
  • GroupMediaPlayerOrangeLight
  • GroupMediaPlayerOrangeDark
  • GroupMediaPlayerPurpleLight
  • GroupMediaPlayerPurpleDark
The tree control does not have any concept of a current style. It actually has a large number of individual properties that control the appearance and operation of the control. When you set a new style using the above method it actually just defines the value of a group of properties to achieve the desired look and feel. So this method is provided to prevent the user from having to spend a large amount of time experimenting to achieve some common look and feel requirements. Think of the method as a shortcut. This technique of using a large number of properties and a shortcut method provide greater flexibility because you are not restricted to having a fixed set of styles. Instead you start from a common style and then tweak the properties as you require.

StandardPlain
As the name implies this is a simple and plain looking appearance. It draws the border, checkboxes, radio buttons and expand/collapse boxes in a simple plain style that ignores the current windows theme. In fact it will look very much like the standard windows tree control where it also ignores the window theme.

StandardThemed
This style has the same appearance as the standard windows tree control when it is taking note of the current windows theme. So the border, checkboxes, radio buttons and expand/collapse boxes are drawn in the theme. Note that you can turn on and off the use of the current theme on an individual basis. So once you set this style you could then modify the border property so that the border is not drawn using the theme but leave the other elements so that they are.

Explorer
The intention of this style is to mimic quite closely the appearance you see in the windows explorer for navigating around folders on your file system. It is also closely related to the StandardThemed style in that it uses the current window theme for drawing the visual elements. You will also notice that it does not draw the connecting lines between nodes and the hot tracking underlines the node text as you move the cursor over nodes.

Navigator
This is the first of two styles that use a different ViewController setting. You can read the following section for a detailed description of the ViewController property and how it works. In summary it means that the root level nodes are drawn and their behavior controlled by a different set of properties to the rest of the nodes that appear below them. The purpose of this style is to make it more visually obvious that the root level nodes represent a grouping of information.

Group
Like the Navigator style, this one also uses the group set of properties for determining the appearance and operation of the root level nodes. This style uses arrows placed on the right hand side of the node in order to indicate if it has children and also if the node is currently expanded or not.

List
As the name implies, this will present the tree structure of nodes in a simple plain list format. There are no connecting lines or expand/collapse boxes either. By default the user is prevented from expanding or collapsing any of the nodes because this would be very confusing when there is no visual indication of which nodes have children. This is a useful style when your data is inherently tree structures but needs to be present as a simple list.

GroupOfficeLight This option is a variation on the existing Group style and will ensure that the group node background is drawn in a similar style to that seen for the tree control in Microsoft Outlook 2003. The background color for the group nodes when selected and not selected will achieve the same color effect.

GroupOfficeDark Another variation on the existing Group style and that also changes the background of group nodes to draw in a dark color that is relevant to the current desktop theme. The darker color is the same as that seen in the title bars that are seen in Microsoft Outlook 2003.

GroupOfficeBlueLight / SilverLight / DarkLight These three related options all provide a Office2007 style appearance in each of the three different color schemes that are provided by the office version. This light version of the colors is appropriate for use when the control is placed inside a docking window content.

GroupOfficeBlueDark / SilverDark / DarkDark These three related options all provide a Office2007 style appearance in each of the three different color schemes that are provided by the office version. This dark version of the colors is appropriate for use when the control is used standalone rather than inside a docking windows content..
ViewControllers
ViewControllers
The possible values for the enumeration property are: -
  • Default
  • Group
The TreeControl itself does not know how to draw individual nodes or how to process actions such as the user clicking the mouse on a node. It acts only as the Model in the traditional Model, View and Controller design pattern. Root level nodes are stored from the TreeControl and a large number of properties exposed but the actual View, Controller sections of the design pattern are implemented in separate classes. In fact a single class is used to implement both the View and the Controller, hence the name of the property is the combination of them both.

Default
As the name suggests, this is the ViewController that is used by default and gives a standard look and feel common to most needs. It applies the node specific properties such as the Font, BackColor, HotBackColor and so forth. Also it will apply the properties that determine the operation of the nodes such as ClickExpand and AutoCollapse.

Group
When you use the SetTreeControlStyle method to define a style of either Navigator or Group then this ViewController is defined for use. It uses an alternative set of properties for determining how the root level nodes are drawn and controlled. This set of group specific properties all begin with the prefix Group, for example GroupBackColor and GroupClickExpand. Nodes that appear below the root level are drawn in the default manner.

Border
This section describes the properties used to control how a border is drawn around the edge of the tree control.

BorderStyle
The possible values for this enumeration property are: -

  • None
  • Theme
  • Solid
  • Dashed
  • Dotted
  • Flat3D
  • Bump3D
  • Etched3D
  • Adjust3D
  • Raised3D
  • RaisedInner3D
  • RaisedOuter3D
  • Sunken3D
  • SunkenInner3D
  • SunkenOuter3D
A value of None will ensure the control does not have any border drawn around the edge of the control. Using the Theme enumeration will draw a border using the currently defined theme on the machine. In this case it will ignore the BorderColor setting as the current theme determines all aspects of the border appearance. If the current machine does not support themes then the control will detect this and instead use the Solid value instead.

If you select a value of Solid, Dashed or Dotted then the value of the BorderColor is used when drawing the border. The remaining group of border values all end in 3D and they do not us the BorderColor to determine the color of the border.

BorderColor
Defines the color to use when drawing the border around the control. Note that this property is only used for BorderStyle values of Solid, Dashed and Dotted.

BorderIndent
This property has four child values called Top, Bottom, Left and Right. Each of these are used to specify the gap between a border edge and the start of drawing for the actual tree nodes. For example, if you define a value of four for the BorderIndent.Top then there will be a four pixel gap between the top border of the control and the start of drawing the nodes inside the control.

Lines
This section describes the properties used to control how lines are drawn between nodes in order to show the parent/child relationship and sibling relationship between nodes in the hierarchy.

ColumnWidth
Space is allocated to the left of a group of nodes in order to draw the connecting lines and boxes that indicate the hierarchical nature of the nodes. If you are using an unusually large or small font for drawing the nodes then you might want to change this width in order to keep the sizing in proportion to the size of the individual nodes.

LineDashStyle
The possible values for this enumeration property are: -

  • Dot
  • Dash
  • Solid
By default a value of Dot is used to give the familiar appearance of most tree controls. But you might want to change this to Dash or Solid to match an alternative look and feel in your application. You should note that changing the LineWidth will alter the appearance of the Dot and Dash lines because the width of the line will alter the length of each individual dot and dash that is drawn.

LineWidth
Use this property to alter the width of the lines used to connect between nodes. Changing the width for styles of Dot and Dash will alter not just the width of the dot or dash but also the length of them. So a width of 1 for a style of Dot will result in alternating pixels of color. If you change the width to 2 then the line will be drawn as a 2 pixel square block followed by a 2 pixel square blank area and so forth.

LineColor
Used to specify the color for drawing the lines.

LineVisibility
The possible values for this enumeration property are: -

  • Nowhere
  • OnlyAtRoot
  • OnlyBelowRoot
  • EveryWhere
If you prefer to have no lines drawn at all then you can assign Nowhere to this property. Using OnlyAtRoot will cause the lines to be drawn at the root level of nodes but not for any child nodes. So the root nodes will have a column on the left with connecting lines for the root siblings. To prevent the root nodes having connecting lines, and so also removing the left column at the root, use the OnlyBelowRoot enumeration value. Alternatively you might want lines drawn at the root and for all child nodes and so use the last and most commonly used value of Everywhere.
Boxes
This section describes the properties used to control how boxes are drawn next to nodes in order to show the parent/child relationship between nodes in the hierarchy. These are the boxes that indicate if a node has children and if the node is currently expanded or collapsed.

ColumnWidth
Space is allocated to the left of a group of nodes in order to draw the connecting lines and boxes that indicate the hierarchical nature of the nodes. If you are using an unusually large or small font for drawing the nodes then you might want to change this width in order to keep the sizing in proportion to the size of the individual nodes.

BoxDrawStyle
The possible values for this enumeration property are: -

  • Plain
  • Themed
  • Gradient
Using the Plain style results in a very simple and flat looking appearance for the expand/collapse boxes. It will draw a square box and fill the inside with a solid color. Any sign that is drawn inside the box is also drawn in a solid color. The Themed option ignores the definition of the colors and BoxLength and instead uses the current theme to determine the appearance. If the machine does not support themes then it reverts back to using the Plain appearance. A rounded appearance is given to boxes when using the Gradient style and the inside color is given a graduated color instead of a solid block.

BoxLength
Defines the width and height of the drawn box. Note that this property is ignored when the BoxDrawStyle is defined as Themed because the theme itself determines the size of the box.

BoxBorderColor
Specifies the color used to draw the box outline. Note that this property is ignored when the BoxDrawStyle is defined as Themed because the theme itself determines the inside color of the box.

BoxInsideColor
Specifies the color used to fill the inside of the box. Note that this property is ignored when the BoxDrawStyle is defined as Themed because the theme itself determines the color of the box.

BoxSignColor
Specifies the color used for any sign drawn inside of the box. Note that this property is ignored when the BoxDrawStyle is defined as Themed because the theme itself determines the color used for any sign inside the box.

BoxVisibility
The possible values for this enumeration property are: -

  • Nowhere
  • OnlyAtRoot
  • OnlyBelowRoot
  • EveryWhere
If you prefer to have no boxes drawn at all then you can assign Nowhere to this property. Using OnlyAtRoot will cause the boxes to be drawn at the root level of nodes but not for any child nodes. To prevent the root nodes having boxes, and so also removing the left column at the root, use the OnlyBelowRoot enumeration value. Alternatively you might want boxes drawn at the root and for all child nodes and so use the last and most commonly used value of Everywhere.

BoxShownAlways
When this boolean property is defined then a box is drawn for a node even when it does not have any child nodes. Normally a node without children will not have a box drawn next to it. When the box is drawn and the node has no child nodes a dot is drawn inside the box instead of the usual plus or minus sign. Note that if you have defined BoxDrawStyle to be Themed then this property is ignored as themes do not have the ability to draw a box for a node without children.

Checkbox / Radio Buttons
This section describes the properties used to control how checkboxes and radio buttons are to be shown and drawn for individual nodes in the tree. It should be noted that individual node instances have the ability to override the display of a checkbox/radio button and so you should consult the node documentation in conjunction with this section to understand the full functionality.

CheckDrawStyle
The possible values for this enumeration property are: -

  • Plain
  • Themed
  • Gradient
Using the Plain style results in a very simple and flat looking appearance for the checkbox/radio buttons. It will draw a square box for the checkbox and a circle for the radio button. Each will be filled with a solid color. The Themed option ignores the definition of the colors and CheckLength and instead uses the current theme to determine the appearance. If the machine does not support themes then it reverts back to using the Plain appearance. When using the Gradient style the inside color is given a graduated color instead of a solid block.

CheckStates
The possible values for this enumeration property are: -

  • None
  • TwoStateCheck
  • ThreeStateCheck
  • Radio
The enumeration defines the global requirement for each node in the hierarchy. So a value of None will result in all the nodes having no checkbox/radio button unless the node itself has requested a different setting. This applies to all the other styles as well, each node inherits this global value unless they have be overriden to a different setting. A TwoStateCheck setting results in a checkbox being shown that can be either checked or unchecked. The ThreeStateCheck option allows checked, unchecked and mixed states to be used. Use the Radio enumeration to show a radio button that can be either checked or unchecked.

CheckLength
Defines the width and height of the area for drawing the checkbox/radio button. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the size of the area.

CheckGapLeft
Defines the space between the start of the node and the start of the checkbox/radio button drawing.

CheckGapRight
Defines the space between the end of the checkbox/radio button drawing and the rest of the node.

CheckBorderWidth
Specifies the thickness of the border for both the checkbox and the radio button. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the size of the box.

CheckBorderColor
Specifies the color used to draw the border around the checkbox/radio button. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the border color.

CheckInsideColor
Specifies the color used to draw inside the checkbox/radio button. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the inside color.

CheckInsideHotColor
Specifies the color used to draw inside the checkbox/radio button when the mouse is hot tracking over the drawing area. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the inside hot tracking color.

CheckMixedColor
Specifies the color used to draw the mixed state symbol inside the checkbox. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the mixed color.

CheckMixedHotColor
Specifies the color used to draw the mixed state symbol inside the checkbox when the mouse is hot tracking over the drawing area. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the mixed hot tracking color.

CheckTickColor
Specifies the color used to draw the checked state symbol inside the checkbox/radio button. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the tick color.

CheckTickHotColor
Specifies the color used to draw the checked state symbol inside the checkbox/radio button when the mouse is hot tracking over the drawing area. Note that this property is ignored when the CheckDrawStyle is defined as Themed because the theme itself determines the tick hot tracking color.

Scrolling
This section describes the properties used to control the scrolling functionality.

VerticalGranularity
The possible values for this enumeration property are: -

  • Pixel
  • Node
The Node vertical granularity is the same as for traditional tree controls, when the user clicks the scrollbar to move up or down one position then the tree moves up and down a whole node at a time. Using the Pixel option will allow the user to scroll up and down a pixel at a time and so have greater control over the visible portion of the tree. This option is particularly useful if your nodes have a variation in height or are very tall and improves usability of the control.

VerticalScrollBar
The possible values for this enumeration property are: -

  • Never
  • Always
  • WhenNeeded
Use this property to control when the vertical scrollbar is make visible. A value of Never will prevent the scrollbar from being shown to the user, even when there are too many nodes to be displayed. Using Always will ensure the scrollbar is displayed all the time but it will only be enabled when there are too many nodes to display them all at once. Last is the default value of WhenNeeded that hides the scrollbar from view until it is needed.

HorizontalScrollBar
The possible values for this enumeration property are: -

  • Never
  • Always
  • WhenNeeded
Use this property to control when the horizontal scrollbar is made visible. A value of Never will prevent the scrollbar from being shown to the user, even when the nodes stretch too wide for display. Using Always will ensure the scrollbar is displayed all the time but it will only be enabled when the nodes stretch too wide to display them entirely. Last is the default value of WhenNeeded that hides the scrollbar from view until it is needed.

EnableMouseWheel
Specifies if the mouse wheel should be allowed to scroll the nodes up and down.

MouseUnits
This property is only applied when the VerticalGranularity is defined as Pixel. In this case pressing the up and down buttons on the vertical scrollbar will scroll by the number of pixels specified by this property. This property is also used when dragging at the top or bottom of the tree client area to determine how many pixels to auto scroll.

MouseWheelUnits
This property determines how many units to scroll when the mouse wheel is used. So detent of the mouse wheel is multiplied by this property to determine the scrolling distance.

IgnoreEnsureDisplayedH
By default this property is defined as True and so whenever a node is selected the horizontal scrolling will be updated to bring the node into view. However, you can override this using False so that no horizontal scrolling occurs by default and only the vertical occurs.

Images
This section describes the properties used to control the drawing of images for individual nodes in the tree. It should be noted that individual node instances have the ability to override the display images and so you should consult the node documentation in conjunction with this section to understand the full functionality.

ImageList
When an ImageList is defined each node will be allocated space for the drawing of an image of the size defined by the ImageList object. If the node itself has a separate image or icon defined then space allocated will instead be the size required for the image or icon defined at the node level.

ImageIndex
Specifies an image index within the ImageList that should be shown for each node that does not have its own image index defined. Usually each node would have its own index specified so that the node can show an image appropriate to its state. A value of -1 for this property would prevent any image being drawn when the node also has no image index specified.

SelectedImageIndex
Specifies an image index within the ImageList that should be shown for each selected node that does not have its own selected image index defined. Usually each node would have its own index specified so that the node can show an image appropriate to its selected state. A value of -1 for this property would prevent any image being drawn when the node also has no selected image index specified.

ImageGapLeft
Defines the gap before the drawing of any image.

ImageGapRight
Defines the gap after the drawing of the image and before the rest of the node.

Drawing
This section describes the properties used to control the drawing of individual nodes in the tree. It should be noted that individual node instances have the ability to override some of the drawing properties and so you should consult the node documentation in conjunction with this section to understand the full functionality.

BackColor
Defines the background color for the whole tree control as well as the default background color for drawing the node text. An individual node can override the background color for drawing the node text.

ForeColor
Defines the foreground color for drawing the node text. An individual node can override the color for drawing the node text.

Font
Defines the font used for drawing the node text. An individual node can override the font for drawing the node text.

HotBackColor
Specifies the background color for drawing the node when the mouse is hot tracking over the node.

HotForeColor
Specifies the foreground color for drawing the node text when the mouse is hot tracking over the node.

HotFontStyle
Specifies the style to be applied to the text font when the mouse is hot tracking over the node. This property is only applied when the UseHotFontStyle is defined as true.

SelectedBackColor
Specifies the background color for drawing the node when it is in the selected mode.

SelectedForeColor
Specifies the foreground color for drawing the node when it is in the selected mode.

SelectedFontStyle
Specifies the style to be applied to the text font when the node is in the selected mode. This property is only applied when the UseSelectedFontStyle is defined as true.

SelectedNoFocusBackColor
Specifies the background color for a node that is selected but does not have the focus.

UseHotFontStyle
Defines if the HotFontStyle property should be applied when hot tracking nodes.

UseSelectedFontStyle
Defines if the SelectedFontStyle property should be applied for selected nodes.

TextRenderingHint
Defines the quality of text drawing used when drawing the node text.

ExtendToRight
When defined this will cause the width of nodes to be stretched so they all extend to the right hand side of the control. Otherwise the width of a node is determined by the minimum width needed for it to be drawn correctly.

Indicators
The possible values for this enumeration property are: -

  • None
  • AtRoot
  • AtGroup
An indicator is a small image that appears on the same line as a node and is used to indicate some state information about the node. By default the None value is used and so there are no indicators displayed. Using AtRoot will cause a small extra column to appear at the left side of the root nodes. Each node that has an indicator defined will have the appropriate image displayed in this column. If you prefer the indicator shown immediately next to the node rather than always at the root then use AtGroup.
Layout
This section describes the properties used to control layout of individual nodes.

VerticalNodeGap
Specifies the vertical pixel gap between each node in the tree.

MaximumNodeHeight
Specifies the upper limit of the allowed height of each individual node.

MinimumNodeHeight
Specifies the lower limit of the allowed height of each individual node.

ContextMenu
This section describes the properties provided to support context menus. Although there are two properties provided for specifiying context menus you can provide the same context menu instance to both properties if you would the same menu to be used under both circumstances.

ContextMenuNode
Specifies the context menu that should be displayed when the user right clicks on a node.

ContextMenuSpace
Specifies the context menu that should be displayed when the user right clicks on the control outside of any individual node.

Selection
This section describes the properties and methods provided to support the selection of nodes. It should be noted that individual nodes have the ability to override some of the selection capabilities and so you should consult the node documentation in conjunction with this section to understand the full functionality.

SelectMode
The possible values for this enumeration property are: -

  • None
  • Single
  • Multiple
Unlike the standard TreeView that comes with the framework, you have the ability to control many aspects of the selection of nodes. Setting the mode to None prevents the user from selecting any of the nodes using either the mouse or the keyboard. Using Single will allow only a single node to be selected at a time. So selecting a new node will automatically deselect any existing selection. The Multiple setting allows the selection of a group of nodes by using the keyboard SHIFT and CTRL keys as appropriate.

NodesSelectable
A boolean property that decides if standard nodes can be selected or not. This might seem a redundant property when you already have the ability to set the SelectMode to be None. But actually this is useful when you are using the view controllers for groups. In this case the SelectMode applies to all nodes in the tree but this property only applies to the nodes below the root level because the GroupNodesSelectable is used for the root nodes. SelectedNodes
This read only property returns a collection containing references to all the currently selected nodes in the tree. Note that altering the contents of the collection returned will not change the selection in the control. You must call methods on the individual node instances to change the selected state.

SelectedNode
This property is more useful when using the Single selection mode. It returns a single selected node and can also be used to set the selected node. When using the Multiple selection mode it returns just one of the currently selected nodes but does not guarantee which it will be.

DeselectAll()
Use this method to remove the current selection state.

Behavior
This section describes the properties provided to support the behavior of nodes.

ClickExpand
The possible values for this enumeration property are: -

  • None
  • Expand
  • Toggle
The value of this property determines what happens when the user single clicks on a node that has children. When defined with a value of None there is no change to the expanded state of the node. A value of Expand would cause the node to become expanded if it is not already in the expanded state. If defined as Toggle then the expanded state is reversed, so if collapsed the node is expanded and if expanded the node is collapsed.

DoubleClickExpand
The possible values for this enumeration property are: -

  • None
  • Expand
  • Toggle
The value of this property determines what happens when the user double clicks on a node that has children. When defined with a value of None there is no change to the expanded state of the node. A value of Expand would cause the node to become expanded if it is not already in the expanded state. If defined as Toggle then the expanded state is reversed, so if collapsed the node is expanded and if expanded the node is collapsed.

CanUserExpandCollapse
Specifies if the user is allowed to change the expanded state of a node by using either the mouse or the keyboard. This is useful if you want to present the nodes in a simple list, where it is unlikely you would allow the user to expand or collapse nodes. As a list the user would be unable to tell if a node has children or not and so the ability to manually change the expanded state is not applicable.

AutoCollapse
Specifies the action to take on sibling nodes when a node is expanded. When this property is defined as True then any node that is expanded will automatically cause all its sibling nodes to be collapsed so that at any time there is only a single node expanded within the collection of nodes.

InstantUpdate
Specifies if changes in state should be reflected immediately or only at the end of a batch of changes. When defined as false the display is invalidated immediately a change of state occurs, but the display might not be actually redrawn until some time in the future once all changes have been made. If defined as true then the control will redraw the control immediately that any change occurs. Use of this property represents a trade off between visual fidelity and performance. Take an example of adding 1000 nodes to the control by using a loop in your code. When defined as false then all the nodes would be added before the display is redrawn to show the new nodes. When defined as true then the display is redrawn after each node is added.

LabelEdit
Specifies if the node text is allowed to be edited.

Tooltips
Specifies if a tooltip should be displayed when a node is only partially visible. The tooltip shown will be the full name of the node and so allow the user to see the hidden portion. Note that this tooltip is a lower priority than the Infotips property that will in preference show any defined individual Node.Tooltip value.

Infotips
Specifies if the Node.Tooltip value for the node instance should be shown when the user hovers the mouse over the node. This takes precedence over the Tooltips property.

AutoEdit
Specifies if the node should automatically start editing when the user clicks the selected node for a second time and there are no other selected nodes.

FocusNode
Specifies the node that currently has the focus. Note that when the control itself does not have focus the FocusNode will still have a valid value for this property, as it represents the individual node that appears with the focus indication when the TreeControl itself has focus. You can also set a new value into the property to shift the focus to another node.

Others
PathSeparator
Specifies the separator string to use when constructing paths for nodes.

ClientPointToNode
Converts a point from client space coordinates to node space.

ClientRectToNode
Converts a rectangle from client space coordinates to node space.

NodePointToClient
Converts a point from node space coordinates to client space.

NodeRectToClient
Converts a rectangle from node space to client space.

Group Drawing
This section describes the properties used to control the drawing of group nodes in the tree. It should be noted that individual node instances have the ability to override some of the drawing properties and so you should consult the node documentation in conjunction with this section to understand the full functionality.

GroupArrows
Specifies if arrows images should be drawn on the right hand side of group nodes to indicate the expanded state of the node.

GroupBackColor
Defines the background color for drawing group nodes. An individual node can override the background color for drawing the node text.

GroupForeColor
Defines the foreground color for drawing group node text. An individual node can override the color for drawing the node text.

GroupColoring>
This enumeration is used to determine how the backround for a group node is drawn. By default the ControlProperties enumeration value will ensure that the group level color properties are used. You can however select Office2003Light or Office2003Dark in order to override the group color properties and ensure the background reflects the current desktop theme, in the same manner as is seen in Office 2003 applications.

GroupFont
Defines the font used for drawing group node text. An individual node can override the font for drawing the node text.

GroupHotBackColor
Specifies the background color for drawing the group node when the mouse is hot tracking over the node.

GroupHotForeColor
Specifies the foreground color for drawing the group node text when the mouse is hot tracking over the node.

GroupHotFontStyle
Specifies the style to be applied to the text font when the mouse is hot tracking over the group node. This property is only applied when the GroupUseHotFontStyle is defined as true.

GroupSelectedBackColor
Specifies the background color for drawing the group node when it is in the selected mode.

GroupSelectedForeColor
Specifies the foreground color for drawing the group node when it is in the selected mode.

GroupSelectedFontStyle
Specifies the style to be applied to the text font when the group node is in the selected mode. This property is only applied when the GroupUseSelectedFontStyle is defined as true.

GroupSelectedNoFocusBackColor
Specifies the background color for a group node that is selected but does not have the focus.

GroupUseHotFontStyle
Defines if the GroupHotFontStyle property should be applied when hot tracking group nodes.

GroupUseSelectedStyle
Defines if the GroupSelectedFontStyle property should be applied for selected group nodes.

GroupGradientBack
Specifies if the group background should be drawn as a gradient or as a solid block of color. A value of true results in the graduated look and false as a solid block.

GroupGradientAngle
The possible values for this enumeration property are: -

  • None
  • LeftToRight

Drag and Drop

User initiated Drag and Drop

event NodeDrag
The user can initiate a drag and drop operation by right clicking and then moving the mouse a short distance. When this happens the NodeDrag event is fired so that the caller has a chance to cancel the operation. Event arguments provided allow the handler to determine the node that is attempting to start the drag. The handler can allow the operation to continue but can modify the DragDropEffects value that is used.

Incoming Drag and Drop

There are a collection of different events generated during the processing of an incoming drag and drop operation. The tree control is split into different areas and each one has its own set of events. Events that are specific to a group node have the Group prefix, events that are specific to non-group nodes have the Node prefix and events in the remaining client space have the prefix Client.

events ClientDragEnter, NodeDragEnter, GroupDragEnter
When the user drags into a new area of the control the appropriate DragEnter event will be fired. This event can be handled and allows the DragDropEffects value that is returned from the drag event to be modified. By default the value is set to None and so indicates that no drag drop can occur at this point. You can use your event handler to modify this value as appropriate.

events ClientDragOver, NodeDragOver, GroupDragEnter
As the user drags the mouse over the control area a series of DragOver events will be fired. This event can be handled and allows the DragDropEffects value that is returned from the drag event to be modified. By default the value is set to None and so indicates that no drag drop can occur at this point. You can use your event handler to modify this value as appropriate.

events ClientDragLeave, NodeDragLeave, GroupDragLeave
Once the user moves the drag cursor away from an area the final event will be a DragLeave. It does not provide a DragDropEffects that can be changed because the drag cursor is leaving the area and so no drop is possible at this time.

events ClientDragDrop, NodeDragDrop, GroupDragDrop
If the user decides to drop inside a control area then the DragDrop event is fired. When over an individual node then the node reference is provided along with the DragDropEffects value so that the handler can perform an appropriate action. ExpandOnDragHover
When the user is moving the dragging cursor over the tree control nodes you will probably want the node to expand if it has children. Otherwise it would be impossible to gain access to a node that is in the tree hierarchy but not currrently showing because a parent node is collapsed. This property allows you to control if the node is auto expanded when the user hovers the dragging cursor over the node.

GroupExpandOnDragHover
This applies the same functionality as the ExpandOnDragHover property above but is applied only to root level nodes when using the Group setting for the ViewController property.

Events
The events listed in this and the previous section represent the ones of most use to the developer. There are however many more events that are generated to indicate a change in a property value. These individual events are not listed but can easily be calculated by just adding the word Changed to the name of the property, for example SelectModeChanged is generated whenever the SelectMode property changes value.

events BeforeSelect, AfterSelect, AfterDeselect, AfterSelectionChanged
The first of these events BeforeSelect is generated when a node is about to become selected. It provides the handler with a reference to the node that is about to be selected along with the ability to cancel the event. Once the node has become selected the AfterSelect event is generated and provides a reference to the target node. When a node is deselected the AfterDeselect event is fired. Note that you will receive many events if selecting or deselecting a large number of nodes using the SHIFT modifier. In this case you might prefer to use the AfterSelectionChanged event that is only fired at the end of the selection change process and so avoids the need to process so many intermediate events.

events BeforeCheck, AfterCheck
The BeforeCheck event is generated when the checkbox/radio button for a node is about to change state. It provides the handler with a reference to the target node along with the ability to cancel the event. Once the checkbox/radio button state has been changed the AfterCheck event is generated and provides a reference to the target node.

events BeforeExpand, AfterExpand
An BeforeExpand event is generated when a node is about to become expanded. It provides the handler with a reference to the node that is about to be expanded along with the ability to cancel the event. Once the node has become expanded the AfterExpand event is generated and provides a reference to the target node.

events BeforeCollapse, AfterCollapse
The first event BeforeCollapse is generated when a node is about to become collapsed. It provides the handler with a reference to the node that is about to be collapsed along with the ability to cancel the event. Once the node has become collapsed the AfterCollapse event is generated and provides a reference to the target node.

events BeforeLabelEdit, AfterLabelEdit
The BeforeLabelEdit event is generated when the node text is about to be edited. It provides the handler with a reference to the target node, the starting text to be edited and the ability to cancel the event. Once the node text has finished being edited the AfterLabelEdit event is generated. It also provides the node reference, the entered text from editing and the ability to cancel the change.

events ShowContextMenuNode, ShowContextMenuSpace
When the user right clicks on a node the ShowContextMenuNode event is fired so that the handler has a chance to cancel the event if desired. If the event is not cancelled then the ContextMenuNode instance is displayed but by cancelling the event the handler can decide to show their own menu instead. The ShowContextMenuSpace event works in the same way but is fired when the user right clicks on client space outside of any of the nodes and will show the ContextMenuSpace menu if not cancelled.

Methods
Expand()
Use this method to expand all the root level nodes that have children and are not already in the expanded state. Note that there are additional methods and properties on the node class for also handling the expanded state of nodes.

ExpandAll()
This method will expand all nodes in the hierarchy that are visible and have children. Note that there are additional methods and properties on the node class for also handling the expanded state of nodes.

Collapse()
Use this method to collapse all the root level nodes that have children and are already in the expanded state. Note that there are additional methods and properties on the node class for also handling the collapsed state of nodes.

CollapseAll()
This method will collapse all nodes in the hierarchy that are visible and have children. Note that there are additional methods and properties on the node class for also handling the collapsed state of nodes.

EnsureDisplayed(Node n)
If you need to ensure that a particular node is in the visible portion of the tree control then use this method. It will adjust the scrolling to bring the node into view if it is currently not showing. Note that if the node is not possible to show because one of its parent nodes is not expanded or if one of its parent nodes is invisible (because the Node.Visible property is false) then the node cannot be displayed. In that case you might want to use the next method instead.

SetTopNode(Node n, bool expand, bool visible)
This method is used to ensure a particular node is used as the first node displayed. If you specify true for the expand parameter then it will expand any parents nodes that are currently collapsed so that the target node becomes visible. Likewise, if you specify true for the visible property then it will make any parent node that has a Node.Visible property of false visible again.

EnsureDisplayed(Node n, bool expand, bool visible)
This method is used to ensure a particular node is visible and allows the caller to specify additional actions to take. If you specify true for the expand parameter then it will expand any parents nodes that are currently collapsed so that the target node becomes visible. Likewise, if you specify true for the visible property then it will make any parent node that has a Node.Visible property of false visible again.

EndEdit(bool quit)
Use this method to cancel the editing of a node label. Providing a parameter of true will cancel the edit and so any changes will be lost. Using false will complete the editing and any changes will be used to alter the text of the node instance.

GetNodeFromKey(object key)
Use this method to find the node instance that has the provided unique key value. This is a useful way to build complex hierarchies using information from a database. You can assign a unique key value to each node instance that relates to a unique field in the database table. Then it becomes easy to find a node instance when you only have the database identifier available.

GetNodeAt() There are three variations on this method. The first takes a single integer parameter and is used to find the Node instance displayed at the requested Y position. It ignores any horizontal positioning of the node and so will match against the node even if the node itself does not cover the entire width of the control. The other two methods take an X and Y value and will find the Node instance that exactly matches the provided coordinate.

GetFirstNode()
Use this method to return the first node in the tree hierarchy. It ignores the expanded or visible state of any node and returns the first node regardless of its current state.

GetFirstDisplayedNode()
Unlike the previous method this one will take into account the expanded and visible state of individual nodes. Therefore it will return the first node in the tree hierarchy that is visible.

GetLastNode()
Use this method to return the last node in the tree hierarchy. It ignores the expanded or visible state of any node and returns the last node regardless of its current state.

GetLastDisplayedNode()
Unlike the previous method this one will take into account the expanded and visible state of individual nodes. Therefore it will return the last node in the tree hierarchy that is visible and will not traverse down a node that is not expanded.

InvalidateNode(Node n)
If you want an individual node to be repainted on the screen then call this method and pass the node reference of interest.

InvalidateNode(Node n, bool recurse)
If you want an individual node to be repainted on the screen then call this method and pass the node reference of interest. You can also specify if you would like all the child nodes to also be invalidated.
