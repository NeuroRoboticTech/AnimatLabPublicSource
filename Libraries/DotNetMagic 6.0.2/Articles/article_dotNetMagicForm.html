DotNetMagicForm Overview
When using the three Office 2007 related visual styles you will want to make use of this base class in order to get the custom chrome that mimics the appearance of Office 2007 applications. It also has the advantage of automatically updating the ToolStrip rendering to match the requested visual style.

Using the base class is very simple, just replace your class inheritance from Form to DotNetMagicForm and then inside your constructor set the Style property to the VisualStyle that you would like the chrome and tool strip rendering to mimic.

There are only two other properties defined by the class, ApplyStyleToChrome and ApplyStyleToToolStrips. As the names suggest, these properties allow you to specify if the base class should determine the window chrome and tool strip rendering based on the Style property.

Appearance

Style property

This property is used to determine the look and feel of the button control. This property is present on most of the DotNetMagic controls and allows a consistent look and feel to be achieved across your whole application. The six possible values for the property are: -
  • VisualStyle.Plain
  • VisualStyle.IDE2005
  • VisualStyle.Office2003
  • VisualStyle.Office2007Blue
  • VisualStyle.Office2007Silver
  • VisualStyle.Office2007Black
  • VisualStyle.MediaPlayerBlue
  • VisualStyle.MediaPlayerOrange
  • VisualStyle.MediaPlayerPurple
VisualStyle.Plain will not draw a border around the button until the mouse moves over the button area and then a border is drawn.

VisualStyle.IDE2005 will draw with the look and feel of the VS.NET 2005 environment.

VisualStyle.Office2003 will draw the button with the same style of gradient effect and colors as the standard Microsoft Office 2003 applications.

VisualStyle.Office2007Blue will draw with the Blue color scheme from Office 2007. VisualStyle.Office2007Silver will draw with the Silver color scheme from Office 2007. VisualStyle.Office2007Black will draw with the Black color scheme from Office 2007. VisualStyle.MediaPlayerBlue will draw in a Blue variation of the Windows Media Player. VisualStyle.MediaPlayerOrange will draw in a Orangevariation of the Windows Media Player. VisualStyle.MediaPlayerPurple will draw in a Purple variation of the Windows Media Player.

ApplyStyleToChrome property

By default this has a value of True and so will alter the window chrome of the form based on the current Style property value. In practice this means that the three Office2007 visual styles will cause custom chrome drawing in the appropriate colors to match the specified style. All other styles result in the window chrome being left to the default of the underlying operating system.

ApplyStyleToToolStrips property

By default this has a value of True and so will alter the global ToolStrip renderer based on the current Style property value. The three Office2007 visual styles will use a custom renderer that applies a gradient effect to the menu items that matches the look and feel of Office 2007 applications. Using the other styles results in the global renderer being defined to one of the standard renderer options that comes with .NET 2.0.
Sample code
C# 

   public partial class MyForm : DotNetMagicForm
   {
      public MyForm()
      {
         InitializeComponent();
         Style = VisualStyle.Office2007Blue;
      }
   }
VB.NET 

   Partial Class MyForm
       Inherits DotNetMagicForm

    Public Sub New()
        InitializeComponent()
		Style = VisualStyle.Office2007Blue
    End Sub
