Please ensure you read all the following sections before using the library in your applications. It is also recommended that you run the samples and read the associated articles before working with any of the controls. This will give you a better feel for the capabilities of the library and the most effective way of using the controls.
Installation
On completing the installation process you will find the following contents have been installed at the location you specified during the install process: -

Articles

This directory contains HTML documents that explain how the various controls that form the DotNetMagic library should be used. When using one of the controls for the first time you should read the appropriate articles to ensure you understand how it can be used most effectively.

Bin

The signed build of the library is contained here and called DotNetMagic2005.dll. Also available is a startup application that allows the samples to be started quickly, this is called SampleStartup.exe. All the sample applications are already present in this directory and built from the C# Samples provided solution.

Help

A single file resides in this directory called DotNetMagicHelp.chm and should be double clicked to start accessing the detailed help for the library. It contains the complete list of classes, methods, properties and events with appropriate descriptions.

Licence

A copy of the licence agreement can be found here and is called Install Full Licence.rtf.

Source

All provided source code is contained in the following directories:-

Build instructions

Building the samples

Each sample has been written in C# so you should navigate to the appropriate language version contained below the Source directory. Double click the solution file found and you will be presented with the list of available samples. Right click the sample application of interest and select the "Set as StartUp Project" option. Now select the "Debug \ Start" menu option and the projects will be automatically compiled and the selected sample started.

Building the DotNetMagic source code

Navigate to the Source\DotNetMagic directory and you will find a Visual Studio .NET solution file called DotNetMagic.sln. Double click this solution and you will be presented with a project that includes all the source code for building an unsigned version of DotNetMagic. The output will be placed by default in the Bin directory and given the name DotNetMagic2005Local.dll to distinguish it from the official signed version that is in the same location and called DotNetMagic2005.dll.

Using DotNetMagic in your own applications

Either create a new project or load the existing project of interest and right click the References folder. This will present a dialog box with the .NET tab already selected. Navigate down the list of assemblies in this tab until you find the DotNetMagic entry that should then be selected. Your project now has a reference to the official signed build of DotNetMagic. When you decide to distribute your project you will need to also distribute the signed assembly which is contained in the Bin directory and called DotNetMagic2005.dll. It is recommended that you add the signed version into the GAC during your own installation process.
Support

Email Support

If you need to ask a support question then please send an email to support@dotnetmagic.com and we will do our best to help as quickly as possible. You should provide the following details:-
  • DotNetMagic Version number
  • Company/Individual name
  • Step by step description of the problem
  • Example application showing the problem
Note that you should send the request from the email address that was used to register when you purchased the product. If this is not possible then you must include that email address as part of the details so that we can check it against the database of purchases.
Licence
To review the details of the licence you should look into the Licence directory and open the contained Install Full Licence.rtf file. If you have any questions then please email to licence@dotnetmagic.com for further clarification.
