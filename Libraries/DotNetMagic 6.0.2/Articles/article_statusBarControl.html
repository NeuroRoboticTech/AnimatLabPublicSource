StatusBarControl Overview
Most applications need a StatusBar to convey information to the user. By using the DotNetMagic provided implementation you can ensure visual consistency because it provides a Style attribute. You can set this to VisualStyle.Office2003 for a look similar to that found in the Microsoft Office 2003 applications. Alternatively use one of the three Office2007 style variations to mimic the appearance of the latest Microsoft Office release.

The following sections show how to use the control and describe the more important properties and events for controlling the StatusBarControl. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Appearance

Style property

This property is used to determine the look and feel of the control. This property is present on most of the DotNetMagic controls and allows a consistent look and feel to be achieved across your whole application. The six possible values for the property are: -
  • VisualStyle.Plain
  • VisualStyle.IDE2005
  • VisualStyle.Office2003
  • VisualStyle.Office2007Blue
  • VisualStyle.Office2007Silver
  • VisualStyle.Office2007Black
  • VisualStyle.MediaPlayerBlue
  • VisualStyle.MediaPlayerOrange
  • VisualStyle.MediaPlayerPurple
VisualStyle.Plain will not draw a border around the button until the mouse moves over the button area and then a border is drawn.

VisualStyle.IDE2005 will draw with the look and feel of the VS.NET 2005 environment.

VisualStyle.Office2003 will draw the button with the same style of gradient effect and colors as the standard Microsoft Office 2003 applications.

VisualStyle.Office2007Blue will draw with the Blue color scheme from Office 2007.

VisualStyle.Office2007Silver will draw with the Silver color scheme from Office 2007.

VisualStyle.Office2007Black will draw with the Black color scheme from Office 2007.

VisualStyle.MediaPlayerBlue will draw in a Blue variation of the Windows Media Player. VisualStyle.MediaPlayerOrange will draw in a Orangevariation of the Windows Media Player. VisualStyle.MediaPlayerPurple will draw in a Purple variation of the Windows Media Player.

Office2003GradBack

This property is only used when the VisualStyle.Office2003 setting is defined for the Style property. When defined as False the control background will be a solid color relevant to the current theme. When True the background will be a gradient from a darker color on the left to a lighter color on the right. This is used to match the color change that can be seen with Microsoft 2003 suite applications on the menu bar area.

IDE2005GradBack

This property is only used when the VisualStyle.IDE2005 setting is defined for the Style property. When defined as False the control background will be a solid color relevant to the current theme. When True the background will be a gradient from a darker color on the left to a lighter color on the right.

SizingGrip property

If your application can be resized by the user then you should set this boolean property so that the far right of the control has a grip area. This allows the user to more easily grab and resize the main application window.

Padding

Four properties are provided that allow fine control over the positioning of the status panels within the status bar control. These are called PadLeft, PadRight, PadTop and PadBottom and all default to a value of 1. By chaning these values you can achieve exactly the positioning required.

StatusPanels property

This property is a collection of the StatusPanel instances for display in the control. You can use the usual collection related methods such as Add, Remove, Count and so forth to manipulate its contents by code. Or in the design mode use the collection editor.
StatusPanel
The StatusPanel instance derives from the Panel base class and adds some additional properties to make it easier to use. The advantage of deriving from the Panel is that during design time you can just drag and drop any control you like onto the panel surface.

For example, it is common to need a ProgressBar in your status bar area. So just create a new StatusPanel in the collection editor and then drag and drop the ProgressBar onto the panel of choice. Then set the Dock property of the ProgressBar to be DockStyle.Fill. Finally define the RequestedWidth of the panel and you are finished.

Image property

Image to be shown in the panel area. This is always drawn on the left of any defined Text.

Alignment property

The three possible values for the property are: -
  • Near
  • Center
  • Far
As the names suggest, Near will draw any Image and/or Text aligned to the left hand side of the panel. Using Far will draw aligned against the right hand side and Center in the middle.

AutoSize property

The three possible values for the property are: -
  • None
  • Contents
  • Spring
None will use the value of the RequestedWidth property to determine the width of the panel. Contents will make the panel just big enough to accomodate the Image and/or defined Text. Spring will divide up any spare space after all other panels have been sized. Note that you can have more than one panel defined with Spring, in which case the left over space is divided equally between them.

RequestedWidth property

Define this simple integer property with the requested width you would like the panel to be. This is only used when the AutoSize property is defined with a value of AutoSize.None.

PanelBorder property

The size possible values for the property are: -
  • None
  • Sunken
  • Raised
  • Dotted
  • Dashed
  • Solid
When you set the Style property the value of this property will be defaulted to Solid for VisualStyle.IDE and Sunken for VisualStyle.Plain. You can however set this value to something different to get an unconventional appearance for your panel.
Sample code
This example shows how to create a StatusBarControl instance and add two panels. The first panel will size itself to the contents and the second will fill all the remaining area.

C#

	
   // Create new control instance
   StatusBarControl sbc = new StatusBarControl();
   
   // Create a panel for showing fixed text
   StatusPanel sp1 = new StatusPanel();
   sp1.Text = "FixedText";
   sp1.AutoSize = StatusBarPanelAutoSize.Contents;
   sbc.StatusPanels.Add(sp1);
   
   // Create a panel to take up all remaining space
   StatusPanel sp2 = new StatusPanel();
   sp2.Text = "Springy";
   sp2.AutoSize = StatusBarPanelAutoSize.Spring;
   sbc.StatusPanels.Add(sp2);
   
   // Add status bar to controls collection
   Controls.Add(sbc);
VB.NET
