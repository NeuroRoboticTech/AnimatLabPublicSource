SlidingTitleBar Overview
This control provides a great way for you to make maximum use of limited screen real estate. It combines the TitleBar control with the ability to progressively slide a child control into and out of view. The TitleBar control is used to give a heading that indicates to the user the nature of the feature it provides. Any Control derived class can be used as the child control that slides in and out of view.

A button is provided on the title area that can be used to show or hide display of the child control. When opened the child control is visible the whole time, but when closed the child is hidden until the user hovers the mouse over the title area. It will then slide into view and remain displayed for as long as the mouse is over the title or child control. Once the mouse is moved away the child will slide back into the hidden state.

An example will illustrate how the SlidingTitleBar can improve usability of an application. Imagine a financial application where the user needs to perform calculations, such as adding up a few numbers, on a regular basis. In this case you could place SlidingTitleBar instance at the top of the client area and attach to it a calculater control. During normal operation only the title part of the control is showing and so takes up very little space. When the user decides to perform a calculation they just hover the mouse over the title area for a short time and the calculater slides into view. It will remain in view as the user performs the actual calculation. Once finished all the user needs to do is move the cursor away from the calculator and it will then slide out of view again.

The following sections show how to use the control and describe the more important properties and events for controlling the SlidingTitleBar. This is not however an exhaustive list and so the help document should be consulted for the definitive list of all functionality.

Design time support
To create an instance of the SlidingTitleBar control you just drag the control from the Toolbox and drop onto the surface of the Form you are designing. You can then use the properties exposed in the Properties window to alter the appearance of the title bar and operation of the sliding effect.

In order to provide the content that will appear in the child portion of the control you need to write code to manually manipulate the child area. It would be much better to expose this area as a designable target in the same way that you can drag and drop controls onto a Panel instance. However, due to limitations in the designer capabilities of .NET you cannot expose the child Panel control from the SlidingTitleBar as a designable surface and so we are stuck with using code.

Here is how you would alter the constructor for your Form to add a Button control that takes up the entire child portion of the sliding control: -

C#

   
   public MyForm()
   {
      InitializeComponent();

      // Create a button instance and add into the sliding control
      Button b = new Button();
      b.Dock = DockStyle.Fill;
      slidingControl1.Panel.Controls.Add(b);
   }
VB.NET
   
   Public Sub New()
      InitializeComponent()

      ' Create a button instance and add into the sliding control
      Dim b as New Button()
      b.Dock = DockStyle.Fill
      slidingControl1.Panel.Controls.Add(b)
   End Sub
Properties

SlideSteps

This property determines the number of steps used to transition from hidden to visible or vica versa. If you provide a value of 1 then you completely remove the sliding effect because it no longer has any intermediate steps.

SlideDuration

The sliding duration is the total length of time, in milliseconds, that the sliding effect should take. If you specify 500 milliseconds and define the number of slide steps to be 10 then each individual step should occur at 50 millisecond intervals.

SlideOnHover

Use this property to turn on or off the sliding effect when the mouse hovers over the title bar area. By default this property is enabled and so when the child control is not visible and you hover over the title bar a slide effect is used to bring it back into view. When disabled the only way the user can show the child is by using the button on the title bar.

HoverDelay

When the SlideOnHover property is defined as enabled, the delay between hovering over the title bar and starting the slide effect is the number of milliseconds defined by this property.

Open

Use this property to define if the child area should be visible all the time. When defined as True the child area is shown but the user can remove it by using the button on the title bar. If defined as False then the user can either use the button to make it visible all the time or hover over the title bar so it slides into view. Note that when the child is shown because of hovering this property still returns False.

Edge

This determines at which edge the title bar is displayed relative to the child area. The four possible values for the property are: -
  • TitleEdge.Top
  • TitleEdge.Bottom
  • TitleEdge.Left
  • TitleEdge.Right

Length

To change the size of the title bar you need to define the Length that it is sized from the Edge it is placed against. So when placed at the top edge, the length will specify the height of the title bar but the width is automatically defined by the width of the whole control. When the title bar is placed on the left edge the Length is the width of the title bar control and the height is automatically defined by the height of the whole control.
Methods

CloseUp

Call this method to have the slide out control removed from display when then Open property is defined as True. This is useful if you want to simulate combo box style behaviour, where the user hovers over the title bar area to show a list or grid style contents. Then the user would select an item and expect the slide out control to disppear again. Calling this method will achieve the effect of snapping back the slide out control. Note that it has no effect when Open is defined as True.
Contained TitleBar
The implementation of the SlidingTitleBar contains an instance of a TitleBar control. Almost all the properties of the contained title bar are exposed from the sliding title bar control so that you can customize its appearance. All the properties are the same name except the background and foreground colour which are exposed as TitleBarForeColor and TitleBarBackColor. See the TitleBar article for more information about its properties.
